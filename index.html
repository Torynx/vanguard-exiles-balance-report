Vanguard Exiles Community Balance Report

Simulator-Based Unit Balance Summary

This report summarizes simulated one-zone melee matchup data across the available stat states and asks how globally balanced the units appear across 1v1, 2v2, 3v3, 4v4, and 6v6 repeated-unit fights.

Update note: This is a re-release of the balance dataset after adding the current patch unit stats and rerunning the simulations.

Generation note: The matchup results were generated with a Python-based Vanguard Exiles simulator, organized in the balance spreadsheet, and summarized into this HTML report with agentic AI assistance.

Workbook: Download the sanitized balance workbook (.xlsx).

Current Patch vs. Earlier Main

Compared with Main pre-v0.3.3, the current patch moves in the wrong direction on the headline simulator metrics: Balance Score drops by 4.4 points, playable matchups drop by 0.9 percentage points, stomp matchups rise by 6.7 percentage points, and power spread increases from 0.842 to 0.914. Methodology

The balance framework measures more than average win rate. It looks for broadly playable matchups, fewer extreme stomp outcomes, smaller strongest-to-weakest gaps, and lower unit power dispersion.

Simulation inputs came from the stat-state configurations represented in the workbook. The Python simulator ran repeated-unit one-zone melee fights for each unit pair and team size, producing win-rate matrices that were then converted into the balance metrics below.

Important limitation: This simulation does not cover real-world match scenarios and should only be used as a measuring stick for controlled unit-stat pressure. Rangers are not designed to be used in melee combat, so Ranger results should be read with that context rather than as a full judgment of live-game Ranger usefulness. The model also does not capture every live-game factor such as player decision-making, board state, positioning beyond the modeled setup, or future design goals.

Metric definitions
Metric Calculation Notes
Balance Score Average of 1 - 2 * absolute(winrate - 0.5), multiplied by 100 across unordered pairs. Standard pairwise fairness score adapted to win-rate matrices.
Playable % Percent of unordered matchups with winrate from 0.35 to 0.65. Wider than Close % because not every matchup needs to be exactly even.
Close % Percent of unordered matchups with winrate from 0.45 to 0.55. Strict balance band.
Stomp % Percent of unordered matchups with winrate at 0.10 or lower, or 0.90 or higher. Lower is better.
Power Spread Strongest unit power index minus weakest unit power index within one size block. Lower is better for global unit parity.
Power StdDev Population standard deviation of unit power indices within one size block. Lower is better.
Power Index Average winrate for one unit across all non-mirror opponents. Shown per unit and per team size.
Scaling Delta Unit 6v6 power index minus its 1v1 power index. Large absolute values indicate size sensitivity.

Most Important Matchup Scaling Risks

Large pair ranges mean that the same unit matchup behaves very differently as team size changes. These are dangerous to tune with a single scalar stat change because the same adjustment can help one size band while making another worse.
